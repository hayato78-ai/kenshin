# 健診結果入力システム設計書（GAS版）

## 1. システム概要

### 1.1 目的
- 健診結果の入力・管理を**属人化せず**に運用できるシステムを構築
- Pythonスクリプト依存からの脱却
- Google Workspace内で完結する自動化システム

### 1.2 基本方針
- **ノーコード運用**: 日常操作にプログラミング知識不要
- **自動処理**: CSV取込から判定・所見生成まで自動化
- **レイアウト維持**: 既存template.xlsmのレイアウトを維持してExcel出力

## 3. 判定ロジック仕様

### 3.1 判定区分

| 判定 | 意味 | 総合所見への反映 |
|------|------|-----------------|
| A | 異常なし | 反映しない |
| B | 軽度異常 | カテゴリ別コメント生成 |
| C | 要精密検査 | カテゴリ別コメント生成（優先度高） |
| D | 要治療 | カテゴリ別コメント生成（最優先） |
| E | 治療中 | 必要に応じてコメント |

### 3.2 判定処理フロー

```
検査値入力
    │
    ▼
┌─────────────────────────────────────┐
│ 判定マスタから基準値取得            │
│ ・項目IDで検索                      │
│ ・性別依存項目は性別で分岐          │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│ 範囲判定                            │
│ if (A下限 <= 値 <= A上限) → A       │
│ else if (B下限 <= 値 <= B上限) → B  │
│ else if (C下限 <= 値 <= C上限) → C  │
│ else → D                            │
└─────────────────────────────────────┘
    │
    ▼
判定結果（A/B/C/D）
```

### 3.3 総合判定ロジック

```
全検査項目の判定を集計
    │
    ▼
最も悪い判定を採用:
  ・1つでもD → 総合D
  ・DなしでC → 総合C
  ・C以下なしでB → 総合B
  ・全てA → 総合A
    │
    ▼
総合判定（A/B/C/D）
```

### 3.4 性別依存項目

| 項目 | 男性基準 | 女性基準 |
|------|---------|---------|
| Hb（血色素量） | 13.1-16.3 | 12.1-14.5 |
| 腹囲 | 84.9以下 | 89.9以下 |
| Cr（クレアチニン） | 0.1-1.0 | 0.1-0.7 |

## 4. 所見自動生成仕様

### 4.1 生成ルール

```
1. 各検査項目の判定を取得
2. 判定B以下の項目をカテゴリ別にグループ化
3. 各カテゴリで最も悪い判定を特定
4. 所見テンプレートからコメントを取得
5. カテゴリ別所見を生成
6. 総合所見に結合
```

### 4.2 カテゴリ別検査項目

| カテゴリ | 検査項目 |
|---------|---------|
| 循環器系 | 血圧（収縮期/拡張期）、心電図 |
| 消化器系 | AST、ALT、γ-GTP、ALP、T-Bil、腹部超音波 |
| 代謝系（糖） | FBS、HbA1c |
| 代謝系（脂質） | TC、HDL-C、LDL-C、TG |
| 腎機能 | Cr、BUN、eGFR、尿蛋白、尿潜血 |
| 血液系 | WBC、RBC、Hb、Ht、PLT |
| その他 | UA、BMI、腹囲、視力、聴力 |

## 7. 運用フロー

### 7.1 通常フロー

```
1. CSV配置
   └→ 01_csv/フォルダにBML結果CSVを配置

2. 自動取込（毎時トリガー）
   ├→ CSV解析
   ├→ スプレッドシートにデータ書き込み
   ├→ 判定処理実行
   └→ 所見自動生成

3. AppSheetで確認・編集
   ├→ 氏名・カナの入力
   ├→ 身体測定の入力
   ├→ 所見の確認・編集
   └→ ステータスを「確認待ち」に

4. Excel出力
   ├→ 出力ボタン押下
   ├→ template.xlsmレイアウトで生成
   └→ 02_出力フォルダに保存

5. 完了
   └→ ステータスを「完了」に
```

### 7.2 手入力項目フロー

```

    │
    ▼

    ▼

    │
    ▼
判定自動更新
    │
    ▼

    │
    ▼

    │
    ▼
```

### 10.1 設定変更方法

| 変更内容 | 方法 |
|---------|------|

### 10.2 トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| CSV取込されない | トリガー停止 | GASでトリガー再設定 |
| 判定が出ない | マスタ未設定 | 判定マスタを確認 |
| Excel出力エラー | 権限不足 | フォルダ権限を確認 |
